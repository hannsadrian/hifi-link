## Timers

The Timers tab integrates with backend endpoints:

- GET /timers â€” list active timers
- POST /timers â€” create a persisted timer
- POST /timers/test â€” fire a one-off timer (not persisted)
- DELETE /timer?id=<id> â€” delete a persisted timer

Configure the device IP/API key under Settings. The app caches the timers list locally and polls every 5 seconds to keep countdowns fresh. Creating a timer computes delay_minutes from the selected type (sleep uses the minutes; wakeup computes minutes until the specified HH:mm, rolling to the next day if necessary).
